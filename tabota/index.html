A workspace for the Tabota notation language. In v2 there is a single recursive type — the Event — and everything in a score is one. An Event is a bag of optional, stackable facets: a frame (the regime role: it turns the Event into a coordinate field its children are read in), a value (a pitch, a lane, an instruction), a position, an extent, and nested events. None are required — an empty {} is a valid Event, and the same facets describe a note, a regime, or a whole movement.

Tabota is a language of description, not scheduling: it can express relations that are indeterminate, over-determined, or even paradoxical. This workspace is a realizer — it plots and compiles the determinate fragment (events placed at exact coordinates with exact or gliding pitch) to microtonal MIDI, retuning through pitch bend so the result travels into a normal DAW. Anything outside that fragment — relational placement (after, meets), ranges, open extents, cycles — stays valid in the language, but is listed under “cannot schedule here” rather than drawn. To draw notes directly, open the piano-roll editor; for rhythm, Cycla treats meter as a recursive subdivision grammar, a tuning system for time.

Compiles one voice — a frame-bearing Event with a pitched axis — to a Standard MIDI File. Each note is placed on the nearest 12-TET key and retuned with a single per-channel pitch bend, so any microtonal frequency transfers exactly. Only the determinate fragment of the voice is written; events positioned by relation, range, or unresolved reference are reported below as unschedulable. The file writes an RPN 0,0 message telling the synth what bend range to expect — set it to match your instrument.
