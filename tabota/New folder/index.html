A read-only window onto the Tabota language. In v2 every node of a score is one recursive Event, a stackable bag of facets — a frame (coordinate field), a value, a position, an extent, nested events. This page and the piano-roll editor read the score through the same interpreter (tabota-resolve.js); they only differ in what each chooses to draw.

Because it never has to play anything, this page draws a wider domain than the Roll: zero-extent points, purely sequential achronous events, indeterminate pitch bands — all things MIDI can't represent. The genuinely unplaceable (relational cycles, indeterminate onsets) can't sit on a timeline, so they're described, not placed below the plot rather than refused. MIDI export lives in the Roll; here you export the visual score.
